"الحين نروح Ads Manager → Events Manager. من فوق يمين اختَر Test Events وحط رابط الموقع. افتح الرابط من هناك—أي events تطلع خلال هالجلسة بتظهر شبه لحظي، وما راح تأثر على بياناتك الفعلية."

Technical steps to fix

StepEnglish (EN)العربية (AR)Notes
Get Pixel ID & base code "First, in Ads Manager go to Events Manager → Pixels. If you don’t have a pixel yet, create one; then copy the pixel ID and the base code snippet." "أول خطوة: من Ads Manager ادخل Events Manager → Pixels. إذا ما عندك Snap Pixel، سوّ Create واحد. بعدها خذ Pixel ID و انسخ الـ base code." Ensure proper access
Choose install path "You have several options to install the pixel – let’s pick the one that’s easiest for you:
• Manual: Paste the base code into the <head> of every page. [guide]
• Google Tag Manager: If you use GTM, you can add the native Snap Pixel tag and set it to fire a 'Page View' event on all pages. [instructions]
• Pixel Set Up Tool (no code): After the base code is in place, you can tag URLs or buttons directly from the tool – great for simple sites. [details]
• Partner integrations: If you're on Shopify or another platform, you can use their Snapchat app. [more]" 		"عندك أكثر من خيار لتركيب Snap Pixel—خلّنا نختار الأسهل والأسرع لكم:
• Manual: نلصق الـ base code داخل <head> بكل الصفحات. [دليل]
• Google Tag Manager (GTM): إذا تستخدم GTM، نضيف Snap Pixel tag ونخليه يرسل PAGE_VIEW على All Pages. [تعليمات]
• Pixel Set Up Tool (no code): بعد تركيب الـ base code، تقدر تسوي tagging للـ URLs أو الأزرار من داخل الأداة—ممتاز للمواقع البسيطة. [تفاصيل]
• Partner integrations: لو منصّتكم مثل Shopify، غالباً فيه تكامل جاهز يسهّل كل شي. [المزيد]" 		Pick the quickest compliant route
Automated Matching "I recommend enabling Automated Matching – it hashes email addresses and phone numbers before sending them to Snap, which helps match more conversions and improves your event quality. It’s also required if you plan to use the Pixel Set Up Tool." "أنصح نفعّل Automated Matching—الفكرة إنه يرسل emails/phones بشكل hashed قبل ما توصل لـ Snap، وهذا يحسّن matching وجودة الـ events. وبرضه مطلوب لو بتستخدم Pixel Set Up Tool." Improves EQS later
Add standard events "Now we need to track key actions – things like 'PAGE_VIEW', 'ADD_CART', 'PURCHASE'. If you’re coding manually or using GTM, make sure the event names are exactly as Snap expects (uppercase). You can also pass parameters like value, currency, and item IDs to power dynamic ads and ROAS reporting." "الخطوة الجاية نحدد أهم actions عندكم ونرسلها كـ standard events زي 'PAGE_VIEW'، 'ADD_CART'، 'PURCHASE'. لو التركيب Manual أو عبر GTM، لازم أسماء الـ events تكون بالضبط وبـ uppercase. وبرضه نضيف parameters مثل value و currency و item_ids عشان ROAS و Dynamic Ads يكونون أدق." Add parameters for value, currency, IDs as needed
Use Pixel Set Up Tool (URL/Button) "If you prefer a no‑code approach, inside the Pixel Set Up Tool you can create URL Events – for example, a rule that fires a 'PURCHASE' event when someone lands on your thank‑you page. Or Button Events that fire when a specific button is clicked (like 'Add to Cart'). Don’t forget to add value and currency for those events if you want to track ROAS." "إذا تبي no‑code، داخل Pixel Set Up Tool تقدر تسوي URL Events—مثلاً يطلق 'PURCHASE' لما العميل يوصل صفحة thank‑you. أو Button Events لما يضغط زر معيّن مثل “Add to Cart”. ولا تنسى تضيف value و currency عشان ROAS يكون صحيح." Fast, no‑code tagging path
Test the setup "After installation, let’s test everything using Test Events – it’ll show you any events fired from your session. Also use the Snap Pixel Helper to double‑check event names and parameters on the live page." "بعد التركيب، نختبر باستخدام Test Events عشان نشوف events اللي تطلع من جلستك. وبعدها نتأكد كمان من Snap Pixel Helper على الصفحة live، ونتحقق من أسماء الـ events والـ parameters." If tests fail → troubleshooting
Improve signal quality "Once events are coming in, check the Event Quality Score in Events Manager. If it’s low, the system will give you recommendations – like passing more customer identifiers or enabling automated matching. Following those tips will help your campaigns perform better." "بعد ما تبدأ الـ events توصل، خلّنا نراجع Event Quality Score (EQS) داخل Events Manager. لو منخفض، بيطلع لك recommendations مثل تفعيل Automated Matching أو تمرير identifiers أكثر. تطبيق هالتوصيات غالباً يرفع performance." Higher EQS → better matching
Optional: Add CAPI (server‑side) "For even more reliable tracking – especially with ad blockers and browser restrictions – I suggest adding the Conversions API (CAPI) alongside your pixel. It sends data directly from your server. We’ll also set up deduplication so you don’t count the same conversion twice." "ولو تبغى تتبّع أدق—خصوصاً مع ad blockers وقيود المتصفح—نقدر نضيف Conversions API (CAPI) جنب Snap Pixel. كذا البيانات تجي server‑side. وبرضه بنفعّل deduplication عشان ما نحسب نفس conversion مرتين." Recommended for scale & privacy

Troubleshooting common issues

IssueEnglish (EN)العربية (AR)Fix
Pixel installed but no events "If the pixel is there but you’re not seeing events, double‑check that your event triggers are set up correctly – for example, a URL rule might need a specific keyword like 'thank-you'. Use Test Events and Pixel Helper to confirm the exact moment the event should fire." "إذا Snap Pixel موجود بس ما نشوف events، غالباً المشكلة في trigger. مثلاً URL rule ممكن يحتاج كلمة معيّنة مثل 'thank-you'. خلّنا نستخدم Test Events و Snap Pixel Helper عشان نحدد بالضبط متى المفروض الحدث يطلق." Check page URL rules or button selector in the tool
Wrong Pixel ID / not shared "Make sure the Pixel ID in your code matches the one in Events Manager. If you have multiple ad accounts or a Business Manager, the pixel might need to be shared with the right account. You can manage sharing in Business Manager settings." "تأكد إن Pixel ID اللي بالكود هو نفسه اللي في Events Manager. وإذا عندكم أكثر من ad account أو Business Manager، ممكن Snap Pixel يحتاج يكون shared للحساب الصحيح. تقدر تعدّل المشاركة من إعدادات Business Manager." Share/unshare in Business Manager as needed
Ad blockers / CSP / script not loading "Sometimes ad blockers or content security policies prevent the pixel from loading. Try testing in an incognito window with extensions disabled. You can also open Developer Tools → Network and look for scevent.min.js – if it’s blocked, you may need to allow sc-static.net in your CSP." "أحياناً ad blockers أو Content Security Policy تمنع تحميل السكربت. جرّب Incognito مع تعطيل الإضافات. ومن Developer Tools → Network ابحث عن scevent.min.js—إذا كان blocked، غالباً تحتاج تسمح للدومين sc-static.net داخل الـ CSP." Re‑test with Pixel Helper
Duplicate events "If you see the same event twice (e.g., from both pixel and CAPI), we need to enable deduplication using event_id. Snapchat uses this ID to match and remove duplicates. Also avoid having overlapping URL rules and button rules that could fire the same event." "لو تشوف نفس event مرتين (مثلاً من Snap Pixel ومن CAPI)، لازم نفعّل deduplication باستخدام event_id. Snap يستخدم هالـ ID عشان يدمج الأحداث ويشيل التكرار. وبرضه نتأكد ما فيه URL rules و Button rules متداخلة تطلق نفس الحدث." Use Events Manager → guidance on dedup
Event syntax / casing "Standard event names are case‑sensitive – for example, you must write PURCHASE (all caps) not purchase. If you use a different casing, Snap won’t recognise it as a standard event." "الـ standard event names حسّاسة لحالة الأحرف: لازم PURCHASE (uppercase) مو purchase. إذا تغيّرت الكتابة، Snap ما راح يتعرّف عليه كـ standard event." Correct naming and re-test
Low signal quality "If your Event Quality Score is low, it means Snap isn’t getting enough reliable data from your events. The fix is usually to enable Automated Matching, pass more customer identifiers (like email or phone), and follow the recommendations in Events Manager." "إذا EQS منخفض، يعني Snap ما قاعد يستقبل signals كفاية عشان يعمل matching قوي. الحل غالباً: تفعيل Automated Matching، وتمرير identifiers أكثر (email/phone)، واتباع recommendations داخل Events Manager." Follow EQS recommendations
Pixel not on all pages "For accurate tracking, the pixel needs to load on every page. If you’re using a CMS, make sure the base code is in your global header or footer. With GTM, ensure the tag fires on all pages, not just a few." "عشان القياس يكون مضبوط، Snap Pixel لازم يشتغل بكل الصفحات. إذا عندكم CMS، خلّوا الـ base code في الـ global header/footer. ومع GTM تأكد إن التاغ Fire على All Pages مو على صفحات محددة." Check theme/header includes
